Blog 2 - Transform and Import Excel data to R

Organizations large and small have begun migrating unstructured Excel data to structured tables to glean valuable insight and improve efficiency. On many cases, there is quantifiable advantage to working with data in structured (tidy) format vs unstructured (untidy) format. The motivation for this blog is two fold. The first is to show that “untidy” Excel data conversion to “tidy” data is possible with “tidyverse” packages alone.


Read More

In case you the reader aren’t familiar with tidyverse, “Tidyverse is a coherent system of R packages for data manipulation, exploration and visualization that share a common design philosophy.” The second motivation is to show the advantages of utilizing R programming language to transform untidy Excel data.

Motivation

Spreadsheet is one of the most prevalent tool to store and share data. Survey from business and government organization show upwards of 90% still use spreadsheet to analyze data and make decisions. Microsoft Excel is, by far, the leading spreadsheet application used for organization sales forecasting, survey, result of clinical test, project status report, cash flow statements, and invoicing to name a few.

To take advantage of advancement in analytic, organizations have begun transitioning Excel data to tidy data format. Tidy data is a dataset that are arranged with each variable saved in its own column, and each observation saved in its own row. Information organized in a tidy format, as we will see later on, make it easy to pull summary statistics, generate visualization and some of the advanced analytics including Machine Learning and Aritificial Inteligence.

The Challenge
Microsoft released the first modern day Excel in 1985. Over the years, Excel spreadsheets are created for all kinds of businesses including research, clinical trials, statistical summarization, engineering and financial services. Historically, Excel is organized and formatted for human consumption. As such, structure and format of the data was not high on the list when preparing a report with Excel. Every preparer used his own expertise to present the data, depending on the field he is in and his audience.

However, with the advent of relatively cheap powerful computers, and a data science workflow, the average business is enabled to process huge amount of data and extract actionable insights. But before value extraction, a number of steps have to be considred including: 1) The historical Excel data was not prepared with data analytics in mind and, therefore, is in untidy format.
2) To analyze data with modern day compute, the data has to be transformed to tidy format.

This is where R comes in. R isn’t the only programming language to format messy data, python comes to mind. R ecosystem comes with a very powerful tools that convert historical ‘untidy’ Excel data into a tidy format.

Before jumping to “how-to” transform untidy Excel data to tidy format, let us take a look at sample Excel data in the following image. You can see, by hovering the magnifying glass over the four sheets (A, B,C & D), that while the data makes intuitive sense to humans, the fact that data and variable are mixed, columns containing multiple values makes it untidy.

The Opportunity
The good news is extracting and tidying historical Excel data is made possible with R tools (will discuss a use case shortly). It takes some effort to get the data prepared, because every Excel ,possibly , is not organized in standardized format. Once data is transformed to Tidy format, and the coloring/notes/formulas used in the excel spreadsheet quantified with R, a valuable actionable insight can be pulled out with additional powerful set of R libraries.

Transforming the Data

To transrom the data I will use dplyr and tidyr packages which are part of the tidyverse bundle. The following high level outline brakes the steps taken to generate the final tidy data.

1/ Import Excel
2/ Create a commodities look up table
3/ Create a partial filled tidy table
4/ Populate the partial filled table with the look-up table
5/ Generate the final tidy data with additional data carpentry

# 1/ Import the data
cali_agri_export <- read_csv("data/cali_agri_export.csv", 
                             col_names = FALSE, col_types = cols(`2015` = col_number(), 
                                                                 `2016` = col_number()))
# 2/ Create a commodities look up table

df_comd <- cali_agri_export %>%
             select(1:3) %>% 
             na.omit() %>% #remove NA lines
             select(base = 1, yr_2015 = 2, yr_2016 = 3) %>% 
             pivot_longer(cols = yr_2015:yr_2016) %>% 
             mutate_all(tolower) %>% 
             filter(stringr::str_detect(base, "[0-9]+")) %>%   
             select(-value) %>% 
             tidyr::separate(base, sep = "_",c("comdty", "rank")) %>%
             select(comdty, rank) %>%
             distinct()  

# 3/ Create a prtial filled tidy table 

df <- cali_agri_export %>%
             select(1:3) %>% 
             na.omit() %>% #remove NA lines
             select(base = 1, yr_2015 = 2, yr_2016 = 3) %>%
             mutate(group = case_when(str_detect(base, "[0-9]+") ~ "11")) %>% #11 place hoder dummy data
             mutate_each(list(as.numeric), group) %>%
             tidyr::separate(base, sep = "_",c("comdty", "rank")) %>% #split base variable
             tidyr::fill(rank) %>% 
             mutate_at(vars(rank:group), ~replace(., is.na(.), 0))  # replace na to 0


# 4/ Populate the partial filled table with the lookup table
df$comdty_1 <- df_comd$comdty[match(df$rank, df_comd$rank)]

# 5/ Generate the final tidy data with additional data carpentry

df_fin <- df %>% 
           mutate_all(tolower) %>% 
           filter(group == 0) %>% 
           select(country = 1, yr_2015, yr_2016, comdty_1) %>%
           pivot_longer(cols = yr_2015:yr_2016) %>% 
           pivot_wider(names_from = comdty_1, values_from = value) %>%
           #clean up var names
           select_all(~str_replace(., " ", "_")) %>%
           select_all(~str_replace(., ",_", "_")) %>%
           select_all(~str_replace(., "and ", "and_")) %>%
           mutate_each(list(as.numeric), 3:52) %>% 
           mutate_at(vars(3:52), ~replace(., is.na(.), 0))

# Tidy data
df_tidy <- df %>% 
            mutate_all(tolower) %>% 
            filter(group == 0) %>% 
            select(country = 1, yr_2015, yr_2016, comdty_1) %>%
            pivot_longer(cols = yr_2015:yr_2016) %>%
            pivot_wider(names_from = name, values_from = value) %>%
            arrange(desc(yr_2015, yr_2016))

Formualting the Question

The first order of business before starting any analysis is to prepare a set of questions we want to ask our data. The questions should be as detailed as possible, because generic questions tend to not yield actionable responses. In practice, more questions arise once we start digging into the data and know more about what other hidden answers can be extracted. For this blog analysis of mtcars dataset, we will limit the number of question to three for brevity.

Let us set a scenario to help frame the questions:
Suppose you want to purchase a vehicle and walk onto a used car sales lot that has 32 used cars. Each car has its own features such as gas consumption measured by miles per gallon (mpg), number of cylinders (four, six or eight), transmission type (automatic or Manual) etc. In the mtcars dataset there are 11 features listed for each car. To shorten the list to few cars you want to test drive, you have chosen the mpg feature to compare the cars with.

You could use the following three questions to help you select gas efficient car from the 32 cars in the sales lot.
1/ You don’t want to spend too much money on gas, so you want to know “which car has the maximum and which car has the minimum mpg for all the cars in the lot”.
2/ The cars on the lot have either 4, 6 or 8 cylinders. You want to know “what the average mpg for each cylinder group?”
3/ Lastly, you want to know for each cylinder group “which cars are the most fuel efficient - the top 5%”

Now that we have the question on hand, we are ready to examine the data and get the answers we are seeking.

It is assumed the reader has basic understanding of tyidyverse and/or R to follow the code in the scripts, but not required to follow and understand the process. To learn on the tidyverse code go here

Importing and cleaning the data

On a typical analysis, getting, cleaning and transforming data takes up to 80% of the time. This is critical stage for any analysis, because without targeted quality data, the outcome is pretty much useless. The mtcars dataset is already in tidy format - it is already cleaned up and in a table with distinct rows and the same features assembled in each column.

Although the matcars dataset is in tidy format and has no missing values to be concerned about, it is still row and requires further transformation to help answer the questions raised. The dataset is relatively small, it contains 32 observations and 11 features. I used tidyverse and base R functions to get the data ready for analysis.

Transform and Explore the data

Next we will transform the original mtcars dataset using tidyverse and statistical functions. As you will see the result of the transformation produces 4 datasets targeting each question raised earlier. The first dataset help us understand the impact of a car weight on mpg, and the other three target each of the three questions raised individually. I will cover feature relationships and feature selection that helps us understand the data on another blog

library(dplyr)  #load the library for transofrorming data

# A. add the model name as a variaiable
df <- mtcars %>% mutate(car_model = row.names(.))

# B. get the maximum and minimum mpg for all cars
mx_mpg <- df %>% slice(which.max(mpg), which.min(mpg)) %>% mutate_each(list(as.factor), 
    cyl)

# C. Get the avg mpg for each cylinder type
avg_mpg <- df %>% group_by(cyl) %>% summarize(avg_wt = mean(wt), avg_mpg = mean(mpg)) %>% 
    mutate_each(list(as.factor), cyl) %>% mutate(note = paste0(round(avg_mpg), 
    " mpg"))

# D. In each cyl group, cars whose mpg is in the top 5%
top_5 <- df %>% group_by(cyl) %>% filter(mpg > quantile(mpg, probs = 0.95)) %>% 
    ungroup() %>% mutate_each(list(as.factor), cyl)

The first dataset (A - df) simply adds the car model name as a feature. This is important because we want to use the car name to label to data points. The second dataset (B - mx_mpg) extracts the which cars have maximum and which cars have minimum mpg from all the cars. The third dataset (C - avt_mpg) calculates the average mpg for each cylinder group, the cars in the lot have either 4, 6 or 8 cylinders. And the last dataset (D - top_5) extracts the top 5% most efficient cars in each cylinder group.

As you can see in the code, I used dplyr verbs such as (gropu_by, summarize, mutate, filter) and statistical functions such (max, min, mean, quantile) and weaved them together to generate additional four new datasets that target the questions raised. This part of the analysis can be accomplished in many different ways and it is both an art and a science. But generally, it is good practice to brake question to small pieces instead of solving everything wholesale.

Summarizing the result of the analysis

It is clear from the graphic visualization that Toyota Corolla is the most fuel efficient car with 33.9 mpg, and Cadillac Fleetwood is the least efficient with at 10.4 mpg. It is also clear that the heavier the car is the higher the gas consumption. We can also see that, on average, the fewer the number of cylinders in a car, the higher more efficient in its fuel consumption. Toyota Corolla, Hornet 4 Drive and Pontiac Firebird are in the top 5 percentile of each cylinder group.

With this information on hand, you can make data driven decision and pick the car to take for test drive.

Take away

On this blog, we took a row data, transform and applied statistics to summarize and extract the answers to our questions. We generated plots to visualize the result. We focused on single feature (mpg) out of eleven, and small dataset to illustrate the data science process, and how to extract value we are seeking. mtcars data set is fairly small, however, the same process is applied for ‘big-data’ sets with had thousands or millions of features and observations!
